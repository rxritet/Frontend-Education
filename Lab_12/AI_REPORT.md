4. ЧТО ВЫ УЗНАЛИ

1. Понял, как работает useWindowDimensions в отличие от Dimensions.get().
Хук подписывается на изменения размера и автоматически триггерит ре-рендер при повороте — это ключевое преимущество перед статичным Dimensions.get.

2. Научился правильно обрабатывать safe area.
SafeAreaProvider нужен на уровне App, а useSafeAreaInsets даёт точные значения insets. SafeAreaView — более простой вариант, но менее гибкий при кастомных заголовках.

3. Понял логику адаптивных макетов на телефоне и планшете.
Одно условие isTablet = width >= 768 меняет layout с колоночного на двухколоночный grid. Это намного проще CSS media queries, потому что логика прямо в компоненте.

4. Разобрался с Platform.select для теней.
iOS использует shadowColor/shadowOffset/shadowOpacity/shadowRadius, Android — elevation. Через Platform.select можно вынести оба варианта в один объект стилей.
